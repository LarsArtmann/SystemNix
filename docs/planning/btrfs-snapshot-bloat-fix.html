Migrate to Disko Future
Declarative disk management — replaces hardware-configuration.nix + manual subvolume creation

Why Disko?

  • Replaces the ensure-btrfs-cache-subvolumes systemd hack entirely
  • Subvolumes created declaratively during disko install
  • Single source of truth for entire disk layout
  • No more manual scripts, no ordering issues

Migration Steps

  1. Add disko flake input to flake.nix
  2. Create platforms/nixos/hardware/disko.nix declaring the full disk layout: 
    {
  disko.devices = {
    disk.nvme = {
      type = "disk";
      device = "/dev/disk/by-uuid/0b629b65-...";
      content = {
        type = "gpt";
        partitions = {
          ESP = { ... };     # /boot (vfat)
          swap = { ... };    # swap partition
          root = {
            content = {
              type = "btrfs";
              subvolumes = {
                "@" = { mountpoint = "/"; };
                "@cache-home" = { mountpoint = "/home/lars/.cache"; mountOptions = ["compress=zstd" "noatime"]; };
                "@go" = { mountpoint = "/home/lars/go"; mountOptions = ["compress=zstd" "noatime"]; };
                "@npm" = { mountpoint = "/home/lars/.npm"; mountOptions = ["compress=zstd" "noatime"]; };
                "@cargo" = { mountpoint = "/home/lars/.cargo"; mountOptions = ["compress=zstd" "noatime"]; };
                "@snapshots" = { };  # snapshot storage (no mount)
              };
            };
          };
        };
      };
    };
    disk.data = {
      type = "disk";
      device = "/dev/disk/by-uuid/046ea663-...";
      content = {
        type = "btrfs";
        mountpoint = "/data";
        mountOptions = ["compress=zstd:3" "noatime" "ssd"];
      };
    };
  };
}
  3. disko --dry-run --mode format,mount to verify (no changes)
  4. Remove hardware-configuration.nix fileSystems, remove ensure-btrfs-cache-subvolumes service
  5. Remove manual fileSystems entries in snapshots.nix (disko manages them)
  6. just switch

Risks

  • Must perfectly match existing layout or disko format could wipe data
  • Test with --dry-run first
  • Have a rollback plan: BTRFS snapshot before running
  • Consider doing this during a fresh install or when you have a full backup
