05What We Should Improve 🔧

1. Deploy immediately

Every fix is inert until nix run .#deploy runs. This is the highest-leverage action.

2. Two-repo deploy automation

The update-push-relock-rebuild flow for private LarsArtmann dependencies is manual and fragile. Add a nix run .#update-dep <name> script.

3. Post-deploy health assertion

deploy.sh should verify that previously-failed services are now running, not just list systemctl --failed.

4. Automatic reset-failed upstream

NixOS's switch-to-configuration should handle start-limit-hit gracefully. Consider an upstream PR or a system.activationScripts hook.

5. Watchdog for stuck-failed services

Four services were broken for the entire previous uptime. Add a timer that alerts on services stuck in failed state.

6. DiscordSync working tree hygiene

Resolve the 17 uncommitted files: commit them if they are valid, or stash/revert if they are WIP.
