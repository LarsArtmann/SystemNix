02Partially Done

Code exists but is incomplete, untested, or unwired.

15 QML widgets are standalone — NOT wired into DMS bar

The widgets poll their services correctly in isolation, but DMS doesn't know they exist. They need to be either: (a) loaded as DMS plugins, (b) imported into a custom shell.qml, or (c) packaged and injected via QML_IMPORT_PATH. This is the biggest gap.

Waybar still runs in parallel

Intentional for safety, but means: (a) two bars on screen, (b) two notification daemons fighting (Dunst is force-disabled but DMS notification server isn't proven), (c) duplicate widgets. Waybar's 7 shell scripts still exist in waybar.nix.

Lock screen and keybinds not updated

LockScreen.qml exists but niri still spawns swaylock on Mod+Shift+Escape. Swayidle still calls swaylock for before-sleep. Dunst history keybind (Mod+Shift+N) still references dunstctl.

QML widgets are syntactically reasonable but untested

Widgets use Quickshell APIs (Process, StdioCollector, FileView, Singleton) that are correct per the v0.3.0 docs. But no Quickshell runtime has loaded them. Import paths, singleton registration, and DMS plugin compatibility are all unverified.

Catppuccin theme not applied to DMS

DMS supports Catppuccin via registryThemeVariants in its settings, but the quickshell.nix module doesn't set it yet. DMS will use its default theme.

Ports not wired from lib/ports.nix

QML widgets hardcode port numbers (e.g. 9090, 11434). The Nix module should template these from lib/ports.nix per AGENTS.md rules, but this hasn't been done.

04Totally Fucked Up

Nothing is broken — yet

The honest risk is that everything is untested at runtime — the QML widgets might crash on load, DMS might not start properly, or the DMS niri module might have a config conflict we can't see from eval alone. This is "calm before the storm" — not "no storm."
