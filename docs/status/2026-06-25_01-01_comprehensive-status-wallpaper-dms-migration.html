07The One Question I Cannot Answer

Should dynamic theming (matugen) override the static Catppuccin Mocha colorScheme?

Context: DMS's enableDynamicTheming = true installs matugen, which extracts Material You colors from the active wallpaper and regenerates theme files for GTK, Qt, terminals, editors, and browsers on every wallpaper change. Meanwhile, the entire SystemNix codebase is built around a static catppuccin-mocha colorScheme (passed as a specialArg, drives Starship, Fzf, Rofi, Ghostty, Kitty, Foot, Zellij, Homepage, and more).

The conflict: When matugen fires, it overrides GTK/Qt/terminal themes with wallpaper-derived colors. But the Nix-managed colorScheme still pushes Catppuccin Mocha to the same targets. This creates a tug-of-war: Nix deploy resets to Catppuccin, wallpaper change resets to matugen. The user sees flickering theme changes or inconsistent colors across apps.

The question: Do you want to commit fully to DMS dynamic theming (remove the static colorScheme from Nix-managed apps that DMS themes) — or disable matugen (enableDynamicTheming = false, set DMS_DISABLE_MATUGEN=1) and keep Catppuccin Mocha as the single source of truth? This is a design decision only you can make — it affects every themed application on the system.
